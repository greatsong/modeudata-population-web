> 머신러닝의 절반은 "어떤 숫자를 모델에 넣을까(특징 설계)"입니다. 알고리즘보다 이게 더 중요할 때가 많아요.

## 1. 특징(feature)이란

머신러닝 모델은 글자("역삼1동")를 모릅니다. **숫자만** 먹어요.
그래서 각 동네를 대표하는 숫자 목록을 만들어 주는데, 그게 **특징**입니다.
6장에서 만든 "비율 벡터"가 바로 특징의 한 예였어요.

여러 동네의 특징을 쌓으면 **특징 행렬 X** 가 됩니다.

```
        나이0   나이1  ...  나이100
동네1 [ 0.009  0.008  ...  0.0001 ]
동네2 [ 0.011  0.010  ...  0.0000 ]
 ...
동네N [ ...                        ]
        └────── 한 행 = 한 동네, 한 열 = 한 특징 ──────┘
```

## 2. 세 가지 특징 설계

이 교재(와 웹앱)는 같은 데이터로 **세 종류**의 특징을 만듭니다. 무엇을 넣느냐에 따라 군집 결과가 달라져요.

먼저 읍·면·동만 골라 남녀 인구 행렬을 준비합니다.

```python
import numpy as np
leaf = df[df["행정구역"].str.endswith(("동","읍","면"))].reset_index(drop=True)
Ml = leaf[male_cols].to_numpy(float)      # (N, 101) 남자
Fl = leaf[female_cols].to_numpy(float)    # (N, 101) 여자
Tl = Ml + Fl                              # (N, 101) 합
tot = Tl.sum(axis=1, keepdims=True)       # (N, 1)   동네별 총인구
```

### ① 연령별 101차원 — "나이 모양만"

남녀를 합친 나이별 비율. **모양**만 봅니다.

```python
X_age = Tl / np.clip(tot, 1, None)        # (N, 101)
```

### ② 성별·연령별 202차원 — "성별까지 구분"

남자 비율 101개 + 여자 비율 101개. 같은 평균연령이라도 **성비가 다른** 동네(군부대, 여대 주변 등)를 구분합니다.

```python
X_sexage = np.hstack([Ml / tot, Fl / tot])   # (N, 202)
```

### ③ + 인구 규모 — "크기까지 반영"

①·②는 비율이라 **규모(작은 시골 vs 큰 신도시)** 를 못 봅니다.
총인구의 로그값을 한 차원 더 붙여, "구조 + 크기"로 묶을 수 있게 합니다.

```python
size = np.log10(tot + 1)                      # (N, 1)  로그로 압축
X_sexage_size = np.hstack([X_sexage, size])   # (N, 203)
```

> **왜 로그?** 인구는 수백~수십만으로 범위가 너무 넓어요. 로그를 씌우면 그 폭이 부드럽게 줄어
> 한쪽으로 쏠리지 않습니다.

## 3. 왜 표준화(스케일링)가 필요한가

문제가 하나 있습니다. ③에서 비율(0~0.02 같은 작은 값)과 로그인구(약 2~6) 를 **그냥 한 행렬에 섞으면**,
숫자가 큰 로그인구가 거리 계산을 독차지해 버립니다. 비율은 묻혀 버려요.

해결책은 **표준화(standardization)**: 각 열을 "평균 0, 표준편차 1"로 바꿔 **단위를 통일**합니다.

```
표준화된 값 = (원래 값 − 그 열의 평균) / 그 열의 표준편차
```

scikit-learn의 `StandardScaler` 한 줄이면 됩니다.

```python
from sklearn.preprocessing import StandardScaler

X_age_s        = StandardScaler().fit_transform(X_age)
X_sexage_s     = StandardScaler().fit_transform(X_sexage)
X_sexage_size_s= StandardScaler().fit_transform(X_sexage_size)

print(X_sexage_size_s.shape)                 # (N, 203)
print("열 평균≈0:", X_sexage_size_s.mean(0).round(3)[:3])
print("열 표준편차≈1:", X_sexage_size_s.std(0).round(3)[:3])
```

이제 모든 차원이 공평하게 거리에 기여합니다. **표준화는 거리 기반 머신러닝(K-평균·PCA)의 필수 준비운동**이에요.

## 4. 정리: 우리가 만든 재료

| 이름 | 차원 | 무엇을 보나 |
|---|---|---|
| `X_age_s` | 101 | 나이 분포 모양 |
| `X_sexage_s` | 202 | 나이 + 성별 |
| `X_sexage_size_s` | 203 | 나이 + 성별 + 규모 |

이 셋을 다음 장에서 **K-평균**에 넣고 결과가 어떻게 달라지는지 비교합니다.

## 🌐 웹앱에서는?

웹앱의 `cluFeatures(rows, mode, includeSize)` 함수가 정확히 이 세 가지를 만듭니다.
- `mode='age'` → 101차원, `mode='sexage'` → 202차원
- `includeSize` 체크 → `Math.log10(총인구+1)` 한 차원 추가
그리고 `zscore(X)` 함수가 우리의 `StandardScaler` 와 똑같이 **열별 z-점수 표준화**를 합니다.
화면의 "차원(특징)" 선택과 "인구 규모도 차원에 반영" 체크박스가 바로 이 설계예요.
