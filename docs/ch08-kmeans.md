## 1. 드디어 머신러닝

7장에서 만든 특징 행렬 `X`(표준화 완료)를 컴퓨터에게 주며 말합니다.
> "정답은 안 알려줄게. 비슷한 동네끼리 **알아서** K개 무리로 묶어 봐."

정답(라벨)이 없으니 **비지도학습**, 비슷한 것끼리 묶으니 **군집화(clustering)** 입니다.

## 2. K-평균은 어떻게 묶을까 (4컷 만화)

```
① 씨앗 뿌리기 : 군집 중심(centroid) K개를 아무 데나 찍는다.
② 줄 세우기   : 모든 점을 '가장 가까운 중심'의 무리에 배정한다.
③ 중심 옮기기 : 각 무리의 '평균 위치'로 중심을 옮긴다.
④ 반복        : ②③을 중심이 안 움직일 때까지 되풀이한다.
```

"가까운 중심에 모이고 → 중심은 무리 한가운데로 이동"을 반복하면, 자연스럽게 비슷한 점끼리 뭉칩니다.

> 💡 **K-평균++ 초기화**: ①에서 씨앗을 완전 무작위로 뿌리면 결과가 들쭉날쭉해요.
> 서로 멀리 떨어지게 똑똑히 뿌리는 방법이 **k-means++** 이고, scikit-learn의 기본값입니다.

## 3. 코드: 한 줄로 군집화

7장의 `X_sexage_s`(202차원, 표준화 완료)를 4개 군집으로 나눠 봅시다.

```python
from sklearn.cluster import KMeans

K = 4
km = KMeans(n_clusters=K, n_init=10, random_state=42)
labels = km.fit_predict(X_sexage_s)      # 각 동네의 군집 번호(0~3)

leaf["군집"] = labels
print(leaf["군집"].value_counts().sort_index())   # 군집별 동네 수
```

- `n_clusters=K` : 몇 개로 나눌지
- `n_init=10` : 씨앗을 10번 다르게 뿌려 가장 좋은 결과 선택(안정성↑)
- `random_state=42` : 매번 같은 결과가 나오도록 고정

## 4. 세 가지 특징으로 비교하기

7장에서 만든 세 재료가 군집을 어떻게 다르게 만드는지 봅시다.

```python
for nm, X in [("연령101", X_age_s), ("성별연령202", X_sexage_s), ("규모포함203", X_sexage_size_s)]:
    lab = KMeans(n_clusters=4, n_init=10, random_state=42).fit_predict(X)
    print(nm, "→ 군집 크기:", np.bincount(lab).tolist())
```

규모를 넣으면 "큰 신도시끼리", "작은 시골 읍·면끼리" 묶이는 경향이 생깁니다.
무엇을 특징으로 넣느냐가 곧 "무엇을 기준으로 묶을까"를 정하는 것이죠.

## 5. K는 몇 개가 좋을까? — 엘보우 방법

K는 우리가 정해 줘야 합니다. 너무 작으면 뭉뚱그려지고, 너무 크면 잘게 쪼개져요.
**관성(inertia)** — 각 점이 자기 중심에서 떨어진 거리 제곱의 합 — 을 K마다 재서 그래프로 봅니다.

```python
import matplotlib.pyplot as plt
ks = range(2, 11)
inertias = [KMeans(n_clusters=k, n_init=10, random_state=42).fit(X_sexage_s).inertia_ for k in ks]

plt.plot(list(ks), inertias, "o-")
plt.xlabel("군집 수 K"); plt.ylabel("관성(inertia)"); plt.title("엘보우 방법")
plt.show()
```

그래프가 **팔꿈치(elbow)** 처럼 확 꺾이는 지점이 적당한 K입니다.
그 뒤로는 K를 늘려도 별로 좋아지지 않거든요.

> 📏 다른 척도로 **실루엣 점수**(`sklearn.metrics.silhouette_score`)도 있습니다.
> 1에 가까울수록 군집이 또렷하게 나뉜 거예요. 입문 단계에선 엘보우만으로 충분합니다.

## 6. 군집의 정체 들여다보기

각 군집이 "어떤 동네 모임"인지 평균 지표로 살펴봅니다.

```python
summary = leaf.groupby("군집").agg(
    동네수=("행정구역", "size"),
    평균연령=("평균연령", "mean"),
    평균총인구=("총인구", "mean"),
    평균소멸위험=("소멸위험지수", "mean"),
).round(2)
print(summary)
```

예를 들어 "평균연령 50세·소멸위험 0.3"인 군집은 **고령 시골 무리**, "평균연령 38세·인구 많음"은 **젊은 신도시 무리** 식으로 성격이 드러납니다.

> 그런데 202차원을 어떻게 눈으로 볼까요? 사람은 2차원밖에 못 봅니다.

## 🌐 웹앱에서는?

웹앱은 scikit-learn이 없으니 K-평균을 **자바스크립트로 직접** 구현했습니다(`kmeans(X, K)` 함수).
- k-means++ 초기화 → 배정 → 중심 갱신 반복: 우리가 배운 4단계 그대로예요.
