## 1. 데이터는 어디서 오는가

분석의 첫 단추는 **신뢰할 수 있는 데이터**를 구하는 일입니다. 우리나라에는 좋은 공공데이터 창고가 많아요.

| 출처 | 무엇이 있나 | 주소 |
|---|---|---|
| 행정안전부 주민등록 인구통계 | 나이·성별·지역별 인구 (이 교재의 데이터) | jumin.mois.go.kr |
| 공공데이터포털 | 정부·공공기관의 거의 모든 데이터 | data.go.kr |
| 통계청 KOSIS | 인구·경제·사회 통계 | kosis.kr |
| 서울 열린데이터광장 | 서울시 데이터 | data.seoul.go.kr |

## 2. 우리가 쓰는 데이터: 행정안전부 「연령별 인구현황」

이 교재의 `population_2026_05.csv` 는 **2026년 5월 행정안전부 주민등록 인구현황**입니다.

- **행정안전부 주민등록 인구통계** 사이트 → 「연령별 인구현황」 메뉴
- 조회 조건: 전국 / 행정구역 단위 / 1세 단위 / 남녀 구분
- 내려받기: **CSV 파일**

이렇게 받으면 한 행이 하나의 행정구역(시도·시군구·읍면동), 각 열이 "남자 0세", "여자 0세" … 인 표가 됩니다.

## 3. 데이터의 모양 미리 보기

받은 CSV의 구조를 글로 그려 보면 이렇습니다.

```
행정구역,            코드,        남자_0세, 여자_0세, 남자_1세, ... , 남자_100세이상, 여자_100세이상
서울특별시,          1100000000,  23925,    22693,    21262,   ... 
서울특별시 종로구,   1111000000,  ...
서울특별시 종로구 청운효자동, 1111051500, ...
...
```

- **행정구역**: 이름. 띄어쓰기로 시도 > 시군구 > 읍면동 위계가 드러납니다.
- **코드**: 10자리 행정동 코드. 나중에 **지도(위경도)와 연결**하는 열쇠입니다.
- **남자_N세 / 여자_N세**: 그 나이의 인구 수. `0세 ~ 99세`, 그리고 `100세이상` 까지.

> 한 살씩 남녀로 나뉘어 있으니 인구 열만 **101(나이) × 2(성별) = 202개**입니다.
> 이 202개 숫자가 뒤에서 머신러닝의 **특징(feature)** 이 됩니다. 기억해 두세요!

## 4. 위경도 데이터 (지도용)

`dong_centroids.csv` 는 각 행정동의 **중심점 좌표**입니다. 구조는 단순해요.

```
코드,         lat,      lon
1111053000,   37.57411, 126.97014
...
```

`population` 파일의 `코드` 와 이 파일의 `코드` 를 맞붙이면, 모든 동네를 **지도 위 점**으로 찍을 수 있습니다.

## 5. ⚠️ 데이터 윤리: 출처를 밝히세요

공공데이터라도 **출처 표기**는 기본 예의이자 신뢰의 근거입니다.
- 언제 데이터인지 (2026년 5월)
- 어디서 왔는지 (행정안전부 주민등록 인구현황)

이 교재의 웹앱도 화면 맨 아래에 출처를 또렷이 적어 두었습니다. 분석 결과를 공유할 땐 항상 출처를 함께 적으세요.

## 🌐 웹앱에서는?

웹앱은 기본 데이터를 같이 배포하면서도, **다른 달 CSV를 직접 올리는 기능**(📂 다른 달 CSV 올리기)을 제공합니다.
행정안전부에서 다음 달 파일을 받아 그대로 올리면 모든 그래프가 새로 그려지죠.
즉, "수집 → 분석"의 수집 부분을 사용자에게 열어 둔 셈입니다.

## 🧪 직접 해보기
1. 행정안전부 주민등록 인구통계 사이트에 들어가 「연령별 인구현황」 메뉴를 찾아보세요.
2. 우리 동네가 데이터에서 몇 번째 행에 있을지 상상해 보고, 3장에서 직접 찾아봅시다.
3. 만약 "가구 수"나 "세대원 수" 데이터가 더 있다면 어떤 새 질문을 할 수 있을지 적어 보세요.
